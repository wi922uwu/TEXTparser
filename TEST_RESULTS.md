## ⚠️ Обнаруженные проблемы

### 1. Модели PaddleOCR
**Проблема:** Модели загружаются при первом запуске backend контейнера, а не предзагружены в образ.

**Влияние:**
- Первый запуск занимает 5-10 минут
- Требуется интернет при первом запуске
- Backend недоступен во время загрузки

**Решение:**
Модели должны быть загружены заранее и скопированы в Docker образ:
```dockerfile
# В Dockerfile добавить:
COPY models/ /app/models/
ENV PADDLEOCR_HOME=/app/models
```

### 2. Зависимости Python
**Проблема:** Несовместимость версий numpy с PaddleOCR.

**Решение:** Использовать numpy==1.24.3 (исправлено в requirements.txt)

## 📝 Инструкции для запуска

### Первый запуск (с загрузкой моделей)

```bash
# 1. Запустить сервисы
cd docker
docker-compose up -d

# 2. Дождаться загрузки моделей (5-10 минут)
docker-compose logs -f backend

# 3. Проверить статус
docker-compose ps

# 4. Проверить health
curl http://localhost/api/v1/health

# 5. Открыть приложение
# http://localhost
```

### Последующие запуски
```bash
cd docker
docker-compose up -d
# Модели уже загружены в volume, запуск быстрый
```
